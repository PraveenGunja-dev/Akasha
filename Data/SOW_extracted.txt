      Project – Akasha A Cross-Platform Intelligence System 
1. Executive Summary
Akasha, is a cross-platform intelligence system that brings together planning, execution, connectivity, logistics, and procurement data into a single dashboard with AI-driven decision support. Unlike traditional dashboards, Akasha includes an AI-powered chatbot that not only provides visibility but also simulates scenarios, recommends actions, and shows potential outcomes before decisions are made. This enables faster decision-making, better coordination, reduced risk, and more efficient project execution.
2. Background & Strategic Mandate
Project data is currently spread across multiple enterprise platforms, causing delays and inconsistent reporting. This initiative aims to create a connected ecosystem where essential project data moves seamlessly, giving stakeholders timely and clear visibility across all project stages.
3. Objective of the Engagement
The main objective is to eliminate disconnected project execution by integrating planning, site progress, connectivity readiness, DPR reporting, logistics, CAPEX, and SAP financial systems into a centralized, real-time monitoring framework. This will provide leaders with a single view of project health and enable timely operational and financial decisions.

4.3 Data & Integration Systems
Primavera P6
Act as the core planning and scheduling engine, supporting work breakdown, activities, baselines, and tracking planned vs actual progress with deviation alerts.
Manage activity sequencing, milestones, and dependencies, while identifying delays and monitoring the critical path with automated alerts
Enable resource allocation and utilization tracking, highlighting over/under utilization for better planning.
Integrate with Digital DPR for real-time updates and ensure continuous alignment between plan and execution.
Improve overall schedule control, risk visibility, and decision-making.
Transmission Connectivity Dashboard
Track grid connectivity, evacuation capacity, and interconnection milestones for overall transmission visibility.
Use the historical data and analyse past timelines (SCD) to identify delay patterns and improve timeline accuracy.
Generate realistic, data-driven projections and adjust timelines (e.g., 4 → 9 months) considering approvals, dependencies, and execution challenges.
Enable execution-based planning (ECOD) by aligning planning and financial decisions with actual timelines.
Improve forecasting, risk identification, capital alignment, and overall decision-making predictability.
Digital DPR (Daily Progress Report)
Centralized Digital DPR system to capture and monitor daily project progress (replacing Excel)
Data access restricted to only the last 7 days to maintain data integrity; any modification within these last 7 days data requires a mandatory reason
Seamless integration with Primavera P6 to enable real-time synchronization of approved progress data and ensure alignment between planned schedules and actual execution
Support role-based access control (Supervisor, Site PM, PMAG Admin) with capabilities for project assignment and controlled cross-project visibility
Defined approval workflow where Supervisor submits DPR, Site PM approves, and only approved data is pushed to P6
Provide real-time dashboards and project summaries, along with tracking of delays, issues, and dependencies, supported by automated alerts and notifications
Improve data accuracy, reduce manual effort, enhance coordination, and enable better decision-making
Module Tracker
Provide end-to-end material tracking from manufacturing to site, including production status, shipment visibility, and monitoring of ETA and transit delays
Ensure alignment with Primavera P6 schedules, highlighting any mismatch between material availability and planned activities
Integrate with SAP for CAPEX tracking to align material movement with financial planning

9. Acceptance Criteria, KPIs & SLAs
Functional Accuracy & Utility
Dashboard Data Accuracy: Consistent alignment between Primavera P6, Digital DPR, Transmission Connectivity, Module Tracker, and SAP data.
DPR Submission Completeness: Site progress reports submitted through the digital DPR system and tracked via system submission logs.
Alert Trigger Accuracy: Reliable automated alerts for delays, connectivity status, logistics movement, and financial updates, monitored through audit logs.
Critical Path Update Latency: Timely reflection of schedule and milestone updates in Primavera P6 following DPR and connectivity data synchronization, tracked via API timestamps.
Financial Data Reconciliation: Alignment between SAP CAPEX and procurement data with project execution records, validated through finance reconciliation reports.
Scenario Simulation Accuracy: System accurately simulates impact of delays and operational changes across schedule, resources, and procurement 
Recommendation Effectiveness: AI-generated recommendations are relevant, actionable, and aligned with business decisions 
Chatbot Interaction Quality: Users are able to query project data and receive clear, context-aware insights
Finance related query- User would be able to get the revenue related answers.
Performance
Dashboard Performance: Fast and responsive dashboard experience for project views.
API Performance: Efficient and stable API communication across integrated systems.
Data Pipeline Timeliness: Near real-time data synchronization across Primavera P6, Digital DPR, Transmission Connectivity, Module Tracker, and SAP
Batch Processing Efficiency: Timely completion of scheduled data processing and system jobs 
Chatbot Response Time: Responses generated within acceptable time for real-time interaction 
Scenario Processing Time: Simulation outputs generated within defined time limits
Security & Compliance
Access Control Enforcement: Role-based access applied across all user personas with periodic access reviews and audit validation.
Security Testing & Hardening: Platform undergoes security assessments before go-live to identify and remediate vulnerabilities.
Audit & Activity Logging: End-to-end logging of user actions, data changes, approvals, and cross-platform transactions for traceability.
API Security Management: Secure API usage across integrated systems with hidden credentials, controlled access tokens, and periodic key rotation.
Integration Security Controls: Protected data exchange between Primavera P6, DPR, Transmission Connectivity, Logistics, and SAP through encrypted channels and secure gateways.
Secure AI Interaction: Chatbot access and responses restricted based on user roles and permissions 
Data Privacy in AI Processing: Sensitive project and financial data handled securely within AI workflows 
Auditability of AI Actions: All chatbot queries, recommendations, and scenario outputs logged for traceability
 Reliability
Platform Stability – The integration platform should always be available and working without interruptions.
Integration Reliability – The platform keeps data in sync across all connected systems. APIs work reliably and keep data accurate. If a data transfer fails, the system automatically retries to keep information flowing smoothly.
Data Reliability – The platform prevents duplicate, incomplete, or conflicting data updates. It also makes sure updates across systems are completed correctly.
Fault Handling & Recovery – The system can detect errors, send alerts, and follow clear steps to fix issues quickly.
Monitoring– The platform continuously monitors system health and data integrations. System logs help teams quickly find and fix problems.
AI System Stability: Chatbot and scenario engine remain consistently available without failures
Adoption
Reporting Reduction: Reduced manual reporting and follow-ups through automated dashboards and integrated data flows.
Decision Support Adoption: Teams refer to AI recommendations during planning and execution decisions
User Satisfaction: Positive user experience and acceptance across leadership and project teams, assessed through periodic feedback and usage reviews.
